3. URL BASE DO PROJETO
- https://fit-body-pro-one.vercel.app

4. ROTAS INTERNAS DO PROJETO
- Criar pagamento: /api/create-payment
- Checar pagamento: /api/check-payment
- Webhook: /api/webhook

5. ENDPOINTS DO MERCADO PAGO
- Checkout Pro: https://api.mercadopago.com/checkout/preferences
- Criar pagamento Pix: https://api.mercadopago.com/v1/payments
- Consultar pagamento: https://api.mercadopago.com/v1/payments/{paymentId}

6. CABECALHOS USADOS
- Authorization: Bearer <MERCADO_PAGO_ACCESS_TOKEN>
- Content-Type: application/json
- X-Idempotency-Key: <valor-unico>  (usado no Pix)

14. CHECAGEM DE PAGAMENTO
Entrada:
{
  "paymentId": "ID_DO_PAGAMENTO"
}

Saida esperada:
{
  "approved": true,
  "status": "approved",
  "data": {}
}

15. FLUXO ATUAL
- Plano free: cadastro direto
- Plano pago: cria pagamento
- Se for Checkout Pro: redireciona para o Mercado Pago
- Se for Pix: gera QR Code e Pix copia e cola
- Front faz polling em /api/check-payment
- Quando approved = true, redireciona para success.html
- Em success.html o usuario cria a conta

16. PONTOS IMPORTANTES PARA INTEGRAR EM OUTRO CODIGO
- Levar create-payment
- Levar check-payment
- Levar webhook
- Configurar MERCADO_PAGO_ACCESS_TOKEN
- Ajustar baseUrl para o dominio novo
- Ajustar success/failure/pending URLs
- Ajustar notification_url do webhook
